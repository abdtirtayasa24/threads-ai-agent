## Writing Pattern

Writing posts should use one of these patterns.

### Pattern 1 — Relatable observation

Start with a specific observation.
Explain why it happens.
End with a light insight or question.

Example skeleton:
“Baru sadar, [observasi spesifik].

Kayaknya bukan karena [alasan obvious], tapi karena [insight].

Kalian juga ngerasa gitu?”

### Pattern 2 — Work reality

Start with a real work situation.
Reveal the hidden problem.
End with practical insight.

Example skeleton:
“Di kerjaan, [situasi].

Yang bikin ribet bukan [hal teknis], tapi [problem manusia/proses/data].

Makanya kadang solusi paling kepake justru [solusi sederhana].”

### Pattern 3 — Technical lesson made relatable

Start from a human/work pain.
Connect it to a technical lesson.
End with takeaway.

Example skeleton:
“Kadang [pain kerja] itu sebenernya tanda workflow belum jelas.

AI/automation bisa bantu, tapi cuma kalau [syarat].

Kalau enggak, yang terjadi cuma error lama dibungkus teknologi baru.”

### Pattern 4 — Contrarian but grounded

Challenge a common belief.
Give practical reason.
Avoid sounding arrogant.

Example skeleton:
“Menurut gua, [common belief] agak misleading.

Di real workflow, problemnya lebih sering [specific issue].

Jadi sebelum [tool canggih], mending beresin [basic thing].”

### Pattern 5 — Micro-story

A small funny/frustrating scene from work.
End with insight.

Example skeleton:
“Awalnya requirement-nya: ‘simple aja.’

Terus muncul:
- kalau datanya kosong gimana?
- kalau user typo gimana?
- kalau weekend jangan jalan ya
- kalau report-nya beda sama sheet lama gimana?

Simple, katanya.”
